# Chat API 内部封装机制详解

## 一、Chat vs Completion

Chat API 只是对 prompt 的结构化封装。

------------------------------------------------------------------------

## 二、角色结构

system\
user\
assistant

底层会转换为特殊 token：

\<\|system\|\>\
\<\|user\|\>\
\<\|assistant\|\>

------------------------------------------------------------------------

## 三、完整输入格式

最终拼接：

\<\|system\|\>系统提示\
\<\|user\|\>用户输入\
\<\|assistant\|\>

------------------------------------------------------------------------

## 四、Function Calling

模型输出结构化 JSON。

通过 schema 约束，提升稳定性。

------------------------------------------------------------------------

## 五、工程注意

-   控制 context 长度
-   明确 system prompt
-   限制最大 token
-   加重试机制
